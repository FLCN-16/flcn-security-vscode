# Security Policy

## Supported Versions

| Version | Supported |
|---|---|
| 1.6.x | Yes |
| < 1.6 | No |

## Reporting a Vulnerability

**Do not open a public GitHub issue for security vulnerabilities.**

Report privately via GitHub's built-in security advisories:
1. Go to the [Security tab](https://github.com/FLCN-16/flcn-security-vscode/security/advisories/new)
2. Click **New draft security advisory**
3. Fill in the details and submit

Or email: **officialr.kay@gmail.com** with subject `[SECURITY] flcn-security-vscode`.

### What to include

- Description of the vulnerability
- Steps to reproduce
- Affected versions
- Potential impact
- Suggested fix (optional)

### Response timeline

- **Acknowledgement** within 48 hours
- **Assessment** within 5 business days
- **Fix & disclosure** coordinated with reporter

We follow [responsible disclosure](https://en.wikipedia.org/wiki/Coordinated_vulnerability_disclosure) — please give us reasonable time to patch before public disclosure.
